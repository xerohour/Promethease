# HTML Style Guide

## Structure
- Use semantic HTML elements whenever possible.
- Keep heading hierarchy valid (`h1` -> `h2` -> `h3`).
- Prefer descriptive `aria-label` text for interactive controls when needed.

## Formatting
- Use 2-space indentation.
- Keep attributes readable; split long tags across lines.
- Avoid inline styles unless unavoidable for generated compatibility.

## Compatibility
- Preserve legacy-compatible markup behavior for existing report pages.
- Avoid broad markup rewrites in generated report artifacts.
