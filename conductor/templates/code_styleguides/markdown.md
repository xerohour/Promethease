# Markdown Style Guide

## Structure
- Use clear headings and short sections.
- Prefer actionable language and user-visible impact first.
- Keep changelog/release notes concise and scannable.

## Formatting
- Use bullet lists for related points.
- Keep file names date-scoped when relevant (e.g., `RELEASE_NOTES_YYYY-MM-DD.md`).
- Use fenced code blocks for commands/examples.
