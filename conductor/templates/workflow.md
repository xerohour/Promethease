# Workflow

## Default Development Workflow
- Required automated/manual validation target: 80% of changed behavior paths must be verified.
- Commit cadence: commit after each task.
- Task summary recording: use Git Notes.

## Quality Expectations
- Keep changes scoped and reviewable.
- Preserve backward compatibility unless change is explicitly approved.
- Validate key user flows on desktop and mobile for UI changes.

## Phase Completion Verification and Checkpointing Protocol
For each phase in a track plan:
1. Run/perform the phase-level validation checks.
2. Summarize outcomes and residual risks.
3. Add/update a checkpoint note before moving to next phase.
